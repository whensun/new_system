## 2.3.1. Reserved Writes Preserve Values, Reads Ignore Values (WPRI)

Some whole read/write fields are reserved for future use. Software should ignore the values read from these fields, and should preserve the values held in these fields when writing values to other fields of the  same  register.  For  forward  compatibility,  implementations  that  do  not  furnish  these  fields  must make them read-only zero. These fields are labeled WPRI in the register descriptions.

<image>

To  simplify  the  software  model,  any  backward-compatible  future  definition  of previously  reserved  fields  within  a  CSR  must  cope  with  the  possibility  that  a  nonatomic  read/modify/write  sequence  is  used  to  update  other  fields  in  the  CSR. Alternatively,  the  original  CSR  definition  must  specify  that  subfields  can  only  be updated atomically, which may require a two-instruction clear bit/set bit sequence in general that can be problematic if intermediate values are not legal.
