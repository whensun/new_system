## 33.1.2. Landing Pad Instruction

When  Zicfilp  is  enabled, LPAD is  the  only  instruction  allowed  to  execute  when  the ELP state  is LP\_EXPECTED .  If  Zicfilp  is  not  enabled  then  the  instruction  is  a  no-op.  If  Zicfilp  is  enabled,  the LPAD instruction causes a software-check exception with x tval set to "landing pad fault (code=2)" if any of the following conditions are true:

- ⚫ The pc is not 4-byte aligned and ELP is LP\_EXPECTED .
- ⚫ The ELP is LP\_EXPECTED and the LPL is not zero and the LPL does not match the expected landing pad label in bits 31:12 of the x7 register.

If a software-check exception is not caused then the ELP is updated to NO\_LP\_EXPECTED .

<image>

The operation of the LPAD instruction is as follows:

```
if (xLPE == 1 && ELP == LP_EXPECTED) // If PC not 4-byte aligned then software-check exception if pc[1:0] != 0 raise software-check exception // If landing pad label not matched -> software-check exception else if (inst.LPL != x7[31:12] && inst.LPL != 0) raise software-check exception else ELP = NO_LP_EXPECTED else no-op endif
```
