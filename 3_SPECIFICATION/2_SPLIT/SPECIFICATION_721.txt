## 3.1.5. Hart ID ( mhartid ) Register

The mhartid CSR is an MXLEN-bit read-only register containing the integer ID of the hardware thread

running  the  code.  This  register  must  be  readable  in  any  implementation.  Hart  IDs  might  not necessarily be numbered contiguously in a multiprocessor system, but at least one hart must have a hart ID of zero. Hart IDs must be unique within the execution environment.

In certain cases, we must ensure exactly one hart runs some code (e.g., at reset), and so require one hart to have a known hart ID of zero.

For  efficiency,  system  implementers  should  aim  to  reduce  the  magnitude  of  the largest hart ID used in a system.
